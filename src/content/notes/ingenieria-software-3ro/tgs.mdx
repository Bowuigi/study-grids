---
title: Teoría General de Sistemas
---

La TGS se aplica a la Ingeniería de Software para gestionar la complejidad, visualizando el software no como código aislado sino como un conjunto de componentes interconectados con un objetivo en común. Permite definir límites, flujos de entrada/salida y jerarquías, asegurando que las partes (módulos, bases de datos, etc) funcionen integralmente.

## Conceptos

- **Sistemas y subsistemas**: Un sistema es una unidad funcional dividida en subsistemas que intercambian información.
- **"E, P, S"**: El software toma datos (Entradas), los procesa o transforma (Procesamiento) y genera un resultado (Salidas).
- **Límites**: Permite definir que forma parte y que no.
- **Interdependencia**: El mal funcionamiento de un componente perjudica a la integridad del sistema completo. Está dada por la cohesión, el acoplamiento y la sinergia.
- **Equifinalidad**: Refiere a la capacidad de un conjunto de sistemas con el mismo objetivo de llegar a soluciones válidas, posiblemente mediante caminos/configuraciones distintas.
- **Homeostasis**: Capacidad de mantenerse estable y prevenir el deterioro o desorden funcional (entropía) mediante el mantenimiento.
- **Sinergia**: La suma de los esfuerzos grupales es mayor que los individuales (también llamado "2+2=5" o "1+1=3")
